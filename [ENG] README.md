This application was developed as part of the Bachelor’s thesis in *Information Systems and Technologies (09.03.02)* at [RTU MIREA](https://english.mirea.ru/), by Alexandr Chekunkov.
The thesis was successfully defended with the grade **“Excellent”**.

---

**Description**

plAnt is a mobile application that allows developers and QA engineers to collaborate on a single platform with the goal of improving the quality of products released by developers.

> The presented system enables developers and testers to interact through a specialized platform functioning as a marketplace for mobile application testing. The system facilitates faster and more efficient defect detection, documentation, and resolution, as well as reducing the time needed to deliver high-quality mobile applications to the market.

---

**Technology Stack**

- Kotlin
- Android SDK
- Kotlin Coroutines, Kotlin Flows
- Jetpack Compose
- Retrofit2
- Dagger2
- JUnit, MockK, Truth

---

**Goals and Objectives**

Goal:
To develop a mobile application for defect detection in applications designed for portable devices.

Objectives:
1. Analyze and compare existing systems for tracking and eliminating defects in applications for portable devices;
2. Design all modules and system components;
3. Develop a defect tracking and resolution system for mobile applications.

---

**Functional and Non-functional Requirements**

| **Functional Requirements**                                                     | **Non-functional Requirements**                                            |
|---------------------------------------------------------------------------------|----------------------------------------------------------------------------|
| Mobile application for the Android OS;                                          | The application must support API level 16 and above;                    |
| Ability to register and authenticate in the system;                             | Optimization for all types of devices;                                    |
| Role selection (developer or QA);                                               | Modern UI design and ease of use;                               |
| Storage of personal information;                                                | Unique login credentials for registration and authentication;           |
| Ability to create, edit, and view existing projects;                            | Personal data must not be stored on the user's device;                   |
|Ability for testers to view project information;                                 | Use of a secure channel for interaction with the remote server;  |
| Ability for developers to create new projects;                                  | Scalability for future upgrades and enhancements; |
|                                     -                                           | Event logging and monitoring must be implemented.                                               |
